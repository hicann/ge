# ResNet-50 Dynamic Batch Image Classification Sample

## Function Description

Implements multi-batch image classification based on the ONNX ResNet-50 network using **dynamic batch** capability. Unlike Sample 1 (fixed single batch), this sample packs 2 images into one batch for simultaneous inference and outputs each image's Top-5 class results.

## Quick Start

If your environment is ready, run the following commands to reproduce (replace `soc_version` as needed):

```bash
# 1. Create directories and download model and images
mkdir -p model data
wget -O model/resnet50_Opset16.onnx "https://github.com/onnx/models/raw/main/Computer_Vision/resnet50_Opset16_timm/resnet50_Opset16.onnx"
wget -O data/dog1_1024_683.jpg "https://obs-9be7.obs.cn-east-2.myhuaweicloud.com/models/aclsample/dog1_1024_683.jpg"
wget -O data/dog2_1024_683.jpg "https://obs-9be7.obs.cn-east-2.myhuaweicloud.com/models/aclsample/dog2_1024_683.jpg"

# 2. Convert model (dynamic batch: requires input_shape and dynamic_batch_size)
cd model && atc --model=resnet50_Opset16.onnx --framework=5 --output=resnet50_dynamic_batch \
  --soc_version=Ascend910B1 --input_format=NCHW --output_type=FP32 \
  --input_shape="x:-1,3,224,224" --dynamic_batch_size="1,2,4,8" && cd ..

# 3. Preprocess images
cd data && python3 ../scripts/transfer_pic.py && cd ..

# 4. Build and run
bash scripts/build.sh && bash scripts/run.sh
```

## Detailed Steps

### Step 1: Prepare Model

1. Download the ONNX model file to the `model/` directory:

    ```bash
    mkdir -p model
    wget -O model/resnet50_Opset16.onnx \
      "https://github.com/onnx/models/raw/main/Computer_Vision/resnet50_Opset16_timm/resnet50_Opset16.onnx"
    ```

2. Convert the dynamic batch offline model (run atc in the `model/` directory, Atlas A2 series example):

    ```bash
    cd model
    atc --model=resnet50_Opset16.onnx --framework=5 --output=resnet50_dynamic_batch \
      --soc_version=Ascend910B1 --input_format=NCHW --output_type=FP32 \
      --input_shape="x:-1,3,224,224" --dynamic_batch_size="1,2,4,8"
    cd ..
    ```

    | Parameter | Description |
    |-----------|-------------|
    | `--input_shape` | Input shape; set batch dimension to `-1` for dynamic batch |
    | `--dynamic_batch_size` | Supported batch sizes that the model adapts to at runtime |
    | `--framework` | Framework type: 0=Caffe, 1=MindSpore, 3=TensorFlow, 5=ONNX |
    | `--soc_version` | Ascend AI processor version, see [Version List](https://hiascend.com/document/redirect/CannCommunityAtcSocVersion) |

    > If you change the `--output` path, update `omModelPath` in the source code accordingly:
    > ```cpp
    > const char* omModelPath = "../model/resnet50_dynamic_batch.om";
    > ```

### Step 2: Prepare Test Images

1. Download test images to the `data/` directory:

    ```bash
    mkdir -p data
    wget -O data/dog1_1024_683.jpg "https://obs-9be7.obs.cn-east-2.myhuaweicloud.com/models/aclsample/dog1_1024_683.jpg"
    wget -O data/dog2_1024_683.jpg "https://obs-9be7.obs.cn-east-2.myhuaweicloud.com/models/aclsample/dog2_1024_683.jpg"
    ```

2. Run the image preprocessing script in the `data/` directory (the script scans `*.jpg` files in the current directory):

    ```bash
    cd data
    python3 ../scripts/transfer_pic.py
    cd ..
    ```

    This generates 2 `*.bin` files in `data/`.

### Step 3: Build and Run

```bash
bash scripts/build.sh
bash scripts/run.sh
```

## FAQ

**Q: What is the difference from Sample 1?**

Sample 1 uses a fixed single batch and processes one image at a time. This sample converts the model with `--dynamic_batch_size`, enabling runtime multi-image batching for higher throughput.

**Q: How to change the batch size at runtime?**

Modify the source code where images are packed into the batch. The batch size must be one of the values listed in `--dynamic_batch_size` (1/2/4/8).

**Q: transfer_pic.py reports `ModuleNotFoundError: No module named 'PIL'`**

```bash
pip3 install Pillow --user
```

**Q: atc reports model file not found**

Ensure you run the atc command in the `model/` directory, or use the correct relative/absolute path in `--model`.
